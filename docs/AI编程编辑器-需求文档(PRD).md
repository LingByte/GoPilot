# AI 编程编辑器（GoPilot Next）需求文档（PRD）

## 1. 背景与目标
### 1.1 背景
随着大模型在代码生成、理解、重构、测试等场景的能力增强，开发者希望在编辑器内以更低摩擦的方式完成“理解-修改-验证”的闭环，而不仅是把 Chat 工具当作外部窗口。

### 1.2 产品愿景
打造一款面向“AI 辅助编程”的桌面编辑器：
- 以本地项目为中心（Project-first）
- 以可验证的改动为中心（Patch-first，可回滚、可审计）
- 以工程化能力为中心（Run/Test/Lint/Git 集成）

### 1.3 目标
- 提供稳定的代码编辑与项目管理体验（达到日常开发可用）
- 提供 AI 助手能力：理解代码、生成补丁、批量重构、解释与问答
- 提供可控的执行与安全策略（避免无意修改与敏感信息泄露）

### 1.4 非目标（本阶段不做）
- 不做完整 IDE（例如全量调试器、复杂语言服务器管理等，后续迭代）
- 不做云端多人实时协作（可作为远期方向）

## 2. 目标用户与使用场景
### 2.1 目标用户
- 个人开发者 / 全栈工程师
- Go/TS/React 等语言使用者（后续扩展）
- 需要频繁重构、补测试、看不熟的代码库的开发者

### 2.2 核心场景（Top Scenarios）
- 打开一个项目 -> AI 解释项目结构与入口 -> 快速上手
- 在某文件中选中一段代码 -> 让 AI 重构/解释/写注释/补测试
- 通过自然语言提出需求 -> AI 生成多文件补丁 -> 预览 diff -> 一键应用
- 运行/测试失败 -> AI 分析日志并给出修复补丁
- 查找引用/跳转定义 -> AI 辅助定位相关文件与调用链

## 3. 范围与信息架构
### 3.1 模块划分
- 工作区与项目（Workspace/Project）
- 编辑器与文件系统（Editor/File System）
- AI 助手（Chat/Inline/Actions）
- 运行与任务（Run/Test/Terminal/Tasks）
- 版本控制（Git）
- 设置与账号（Settings/Auth，可选）
- 插件与扩展（Extensions，可选）

### 3.2 核心页面
- Home（最近项目、打开/创建项目）
- Editor（主工作台）
- Settings（模型、API Key、安全策略、主题等）

## 4. 功能需求（MVP）

> MVP 标准：能完成“打开项目 -> 编辑代码 -> AI 生成改动 -> 预览 diff -> 应用 -> 运行命令验证”的闭环。

### 4.1 项目与工作区
- 打开本地文件夹作为项目
- 最近项目列表
- 基础项目索引：
  - 文件树展示（折叠/搜索）
  - 过滤（.gitignore/自定义忽略规则）

**验收标准**
- 能在 2 秒内展示 5k 文件规模项目的文件树（允许懒加载）
- 支持按文件名快速搜索并打开

### 4.2 编辑器
- 多标签页（Tab）
- 基础编辑能力：撤销/重做、查找替换、格式化（可先对接外部 formatter）
- 语法高亮（至少支持 TS/JS/Go/JSON/Markdown）

**验收标准**
- 打开 5MB 文件不卡死（可降级）
- 快捷键与基本交互可用

### 4.3 终端与任务
- 内置终端（至少 1 个会话）
- 支持一键执行常用命令（Run / Test / Lint）
- 命令输出可复制、可被 AI 引用

**验收标准**
- 能执行 `npm test`/`go test` 等并完整显示输出
- 任务失败时可将输出一键发送给 AI

### 4.4 AI 助手（核心）
#### 4.4.1 Chat（对话）
- 右侧/底部面板提供 Chat
- 支持将：
  - 当前文件
  - 选中内容
  - 多文件片段（按引用/搜索结果）
  作为上下文发送

#### 4.4.2 Inline（行内建议，可选 MVP+）
- 触发方式：快捷键/输入提示
- 接受/拒绝建议

#### 4.4.3 Actions（可执行的 AI 操作）
- 生成补丁（Patch）而不是直接改文件
- Patch 预览：
  - 以 diff 形式展示
  - 支持逐文件勾选、逐 hunk 勾选
  - 支持一键应用/回滚

**验收标准**
- AI 生成的改动必须以 diff 形式展示，且不经用户确认不得落盘
- 支持最少：单文件补丁 + 多文件补丁的预览与应用

### 4.5 搜索与导航
- 全局搜索（按文件名、按内容）
- 基础跳转（最近打开、定位行号）

**验收标准**
- 10k 文件规模下，文件名搜索 < 200ms（允许索引）

### 4.6 设置
- 模型提供商选择（OpenAI/兼容 API/本地模型，按配置项扩展）
- API Key 管理（不写入仓库，安全存储）
- 上下文策略（最大文件数/最大 token/忽略规则）

**验收标准**
- API Key 不落盘明文（Windows 可用 Credential Manager 或 Tauri secure storage）

## 5. 功能需求（增强/后续）
- Git 面板（状态、diff、提交、分支切换）
- Diagnostics：集成 LSP（Go gopls、TS server）
- 代码索引与引用图（call graph）
- 多模型路由（不同任务用不同模型）
- Agent 模式：
  - 计划（Plan）
  - 执行（Apply patches）
  - 验证（Run tests）
  - 总结（Report）
- 插件系统（可加载自定义 actions/providers）

## 6. 非功能需求
### 6.1 性能
- 冷启动 < 3s（MVP 目标，视机器）
- 打开项目索引可渐进式加载
- AI 请求需有超时与取消

### 6.2 稳定性
- 关键操作有错误提示与可恢复策略
- Patch 应用要么全部成功要么回滚

### 6.3 安全与隐私
- 明确提示哪些内容将发送给模型
- 支持敏感文件规则（例如 `.env`、密钥、证书）默认不发送
- 支持脱敏（可选）

### 6.4 可观测性
- 基础日志（请求耗时、失败原因）
- 本地诊断信息导出（用于反馈）

## 7. 数据与状态
- RecentProjects：最近项目列表（路径、打开时间、显示名）
- WorkspaceSettings：每个项目的设置（忽略规则、默认命令）
- GlobalSettings：模型、Key、安全策略
- ChatHistory：会话记录（可按项目隔离）

## 8. 交互与 UX 约束
- AI 输出必须结构化：
  - 建议/解释/补丁分区
  - 补丁必须可预览
- 关键流程必须可撤销/回滚
- 对于耗时操作（索引、AI 请求、运行命令）提供进度与取消

## 9. 技术约束与建议实现
- 桌面端：Tauri
- 前端：React + TypeScript + Vite
- 编辑器：Monaco 或 CodeMirror（MVP 可先选其一）
- Diff：基于 `diff`/`monaco diff editor`/自研 hunk 选择
- AI：统一的 Provider 接口（OpenAI-compatible）

## 10. 里程碑（建议）
### M0：脚手架与最小闭环（1-2 周）
- Home：打开项目
- Editor：打开/编辑文件
- Terminal：执行命令

### M1：AI Chat + Patch（2-4 周）
- Chat 面板
- 选择上下文
- 生成 diff + 预览 + 应用

### M2：工程化增强（4-8 周）
- Git 面板
- 搜索/索引优化
- 任务模板（run/test/lint presets）

## 11. 验收清单（MVP）
- 能打开一个本地项目并浏览文件树
- 能打开文件并编辑保存
- 能运行 `npm test` 或 `go test` 并展示输出
- 能把错误日志发给 AI，并收到可应用的 diff
- 能预览并应用 diff，且支持撤销/回滚
