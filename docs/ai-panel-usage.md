# AI 面板使用指南

## 🎯 功能概述

我们在 GoPilot 编辑器的右侧添加了一个全新的 **AI Assistant** 面板，提供两大核心功能：

1. **AI 聊天** - 与 AI 助手进行智能对话
2. **任务拆解** - 将复杂需求自动拆解为具体开发任务

## 🚀 如何使用

### 1. 打开 AI 面板

在 GoPilot 编辑器的右侧活动栏中，点击 **🤖 AI Assistant** 按钮：

```
右侧活动栏按钮：
┌─────────────┐
│    📁 Files │
│    🔍 Search│
│    🌿 Git   │
├─────────────┤
│    ⚡ Tools │
│    🗄️ Database│
│    🤖 AI    │  ← 点击这里！
└─────────────┘
```

### 2. AI 聊天功能

#### 切换到聊天模式
- 点击面板顶部的 **"AI 聊天"** 标签页
- 输入您的问题，例如：
  - "如何优化 Rust 代码的性能？"
  - "解释一下 React Hooks 的使用方法"
  - "帮我调试这个 TypeScript 错误"

#### 使用示例
```
用户: 如何在 Rust 中实现一个简单的 HTTP 服务器？

AI: 您可以使用 tokio 和 warp 库来实现。以下是一个简单的示例：

use warp::Filter;

#[tokio::main]
async fn main() {
    // GET /hello/warp => 200 OK with body "Hello, warp!"
    let hello = warp::path("hello")
        .and(warp::path::param::<String>())
        .map(|name| format!("Hello, {}!", name));

    warp::serve(hello)
        .run(([127, 0, 0, 1], 3030))
        .await;
}
```

### 3. 任务拆解功能

#### 切换到任务拆解模式
- 点击面板顶部的 **"任务拆解"** 标签页
- 输入您的需求描述

#### 使用示例
```
输入需求: 创建一个用户管理系统，包括注册、登录、权限管理功能

AI 拆解结果:

📋 任务拆解结果

## 任务 1: 设计功能架构
- **类型**: Design
- **优先级**: High
- **预估时间**: 60 分钟
- **描述**: 设计 创建一个用户管理系统，包括注册、登录、权限管理功能 的架构和实现方案
- **需要文件**: design.md
- **验收标准**: 架构设计完成

## 任务 2: 实现核心功能
- **类型**: Implementation
- **优先级**: High
- **预估时间**: 240 分钟
- **描述**: 实现 创建一个用户管理系统，包括注册、登录、权限管理功能
- **需要文件**: main.rs
- **验收标准**: 功能正常工作

## 任务 3: 编写测试用例
- **类型**: Testing
- **优先级**: Medium
- **预估时间**: 120 分钟
- **描述**: 为 创建一个用户管理系统，包括注册、登录、权限管理功能 编写测试
- **需要文件**: test.rs
- **验收标准**: 测试覆盖率 > 80%
```

## 🎨 界面特性

### 响应式设计
- **最小宽度**: 350px
- **默认宽度**: 420px
- **可调整大小**: 支持拖拽调整面板宽度

### 交互特性
- **实时对话**: 支持连续对话，保持上下文
- **加载状态**: AI 思考时显示动画效果
- **时间戳**: 每条消息都显示发送时间
- **键盘快捷键**: 按 Enter 发送消息，Shift+Enter 换行

### 视觉反馈
- **用户消息**: 蓝色气泡，右对齐
- **AI 消息**: 灰色气泡，左对齐
- **加载动画**: 三个跳动的点

## ⚙️ 技术实现

### 前端组件
- **React + TypeScript**: 现代化的前端框架
- **Tailwind CSS**: 美观的样式设计
- **Lucide React**: 精美的图标库

### 后端集成
- **Tauri Commands**: 安全的前后端通信
- **Rust 后端**: 高性能的任务拆解引擎
- **AI 服务**: 支持 OpenAI、Ollama 等多种 AI 提供商

### 核心命令
```rust
// AI 聊天
invoke('ai_chat', { request: ChatRequest })

// 任务拆解
invoke('analyze_requirement', { requirementText, projectContext })
invoke('simple_decompose_requirement', { requirement })
```

## 🔧 配置要求

### AI 配置
在使用 AI 功能之前，请确保：
1. 配置了有效的 AI 服务提供商
2. 设置了正确的 API 密钥
3. 网络连接正常

### 配置方法
在 Settings 中配置 AI 服务：
```json
{
  "ai": {
    "provider": "openai",
    "api_key": "your-api-key",
    "model": "gpt-3.5-turbo"
  }
}
```
